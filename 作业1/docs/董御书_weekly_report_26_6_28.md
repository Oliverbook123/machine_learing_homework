## 二、学习内容详述

### 2.1 机器学习整体流程

通过首次完整地走通了监督学习的一个完整流程，理解到机器学习项目的基本骨架是：

```
数据加载 → 预处理（归一化、划分训练/验证集） → 定义模型 → 定义损失函数和优化器
→ 训练循环（前向传播 → 计算损失 → 反向传播 → 参数更新） → 验证 → 预测 → 评估
```

### 2.2 数据预处理

此前对数据预处理的认识停留在"缺失值填充"层面。在本项目中学习到两种重要的预处理技术：

| 技术 | 所学内容 | 代码 |
|---|---|---|
| **Z-score 归一化** | 将特征转为均值 0、标准差 1 的分布，公式 `(x - μ) / σ` | `(X - feat_mean) / feat_std` |
| **训练/验证集划分** | 从训练数据中留出一部分做验证，用于在训练过程中评估模型泛化能力 | `np.random.permutation` + 80/20 切分 |

**Z-score 归一化为什么重要？** 因为不同特征的数值范围差异巨大（比如"戴口罩比例"是 0~100，"确诊病例数"可能有上万的量级）。如果不做归一化，数值大的特征会在损失函数中占主导地位，模型会忽略数值小的特征。归一化让所有特征站在同一起跑线上。

### 2.3 PyTorch 核心概念

以下每个模块都是在本周的代码中第一次接触并边写边学的：

#### 2.3.1 张量（Tensor）—— PyTorch 的数据载体

| 函数 | 功能 | 在本项目中的用途 |
|---|---|---|
| `torch.tensor(data, dtype=torch.float32)` | 将 numpy 数组转为 PyTorch 张量 | 将预处理后的 numpy 数据送入模型 |
| `.numpy()` | 将张量转回 numpy 数组 | 将模型预测结果转回 numpy 格式保存 |

张量本质上就是带 GPU 加速能力、能自动求导的多维数组。在此之前我理解的数据结构是 `numpy.ndarray`，现在知道 PyTorch 的一切计算都围绕 Tensor 展开。

#### 2.3.2 网络层（`nn.Module` 子类）

| 层 | 数学含义 | 在本项目中的用途 |
|---|---|---|
| `nn.Linear(in, out)` | 全连接层：`y = xW^T + b`，权重默认 `requires_grad=True` | 每一层的线性变换 |
| `nn.ReLU()` | 激活函数：`f(x) = max(0, x)`，引入非线性 | 让多层叠加不等于单层线性 |
| `nn.BatchNorm1d(dim)` | 批归一化：`(x - μ_batch) / σ_batch * γ + β` | 稳定训练、加速收敛 |
| `nn.Dropout(p)` | 训练时随机丢弃 p% 的神经元 | 正则化，防止过拟合 |
| `nn.Sequential(...)` | 有序容器，按顺序执行各层 | 组织模型结构，省去手写 forward |

**关键理解**：`nn.Linear` 创建的权重参数**默认就自动追踪梯度**（`nn.Parameter(requires_grad=True)`），不需要手动设置。这个知识点来自一次困惑："为什么代码里没有 `requires_grad=True`？"——查阅后才知道 `nn.Parameter` 默认开启梯度追踪。

#### 2.3.3 损失函数与优化器

| 模块 | 功能 | 在本项目中的用途 |
|---|---|---|
| `nn.MSELoss()` | 计算均方误差 `∑(y_pred - y_true)² / n` | 回归任务的损失函数 |
| `optim.Adam(params, lr)` | 自适应学习率的优化算法 | 根据梯度更新模型参数 |
| `optimizer.zero_grad()` | 清空上一轮的梯度缓存 | 防止梯度在上一次的基础上累加 |
| `loss.backward()` | 反向传播，自动计算所有参数的梯度 | 链式求导 |
| `optimizer.step()` | 根据梯度更新参数 | 完成一步梯度下降 |

**Adam 优化器的直观理解**：梯度下降就像下山——每次朝最陡的方向走一步（学习率控制步长）。普通的 SGD 每次都用同样的步长，Adam 会自适应地调整每个参数的步长：陡的地方走小步（怕冲过头），平的地方走大步（加速收敛）。这解释了为什么 Adam 通常比 SGD 更省心——不需要手动调学习率衰减策略。

#### 2.3.4 数据加载

| 模块 | 功能 |
|---|---|
| `TensorDataset(X, y)` | 将特征和标签打包成数据集 |
| `DataLoader(dataset, batch_size, shuffle)` | 按 batch 切分数据，shuffle=True 时每轮打乱 |

**为什么需要 DataLoader？** 因为训练时如果一次性把所有 2160 个样本都喂给模型（batch_size=2160），内存和显存压力大，且梯度估计的方差小容易陷入局部最优。分成小 batch（64 个一批）不仅能缓解内存压力，mini-batch 的随机性本身也起到一定的正则化作用。


### 2.4 归一化的两个层次

通过本项目的实践，深入理解了两种归一化的区别与联系：

| 维度 | 特征级 Z-score 归一化 | Batch Normalization |
|---|---|---|
| **作用位置** | 数据预处理阶段 | 网络内部每一层 |
| **作用对象** | 原始输入特征 X | 每层 Linear 后的激活输出 |
| **统计量来源** | 整个训练集的 μ/σ（离线计算） | 当前 mini-batch 的 μ/σ（在线估计） |
| **学习能力** | 固定的线性变换 | 带有可学习的 γ（scale）和 β（shift） |
| **实现** | `numpy` 手动计算 | `nn.BatchNorm1d` 层 |

**关键认知**：两者**不是替代关系，而是互补关系**。输入特征归一化保证数据进入网络时尺度统一，BN 解决的是数据流过各层后尺度漂移的问题（内部协变量偏移）。在实验中，数据已经做了 Z-score 归一化，但 BN 仍然有微小的正收益（验证 RMSE 从 1.0074 降到 0.9965）。

### 2.5 模型容量与表达能力

| 模型 | 参数量 | 数学表达 | 最佳验证 RMSE |
|---|---|---|---|
| 线性回归 | 94 | `y = Σ w_i·x_i + b` | 7.29 |
| MLP | 20,353 | 多层非线性变换 | 0.98 |

**从线性到非线性的跨越不是量变，是质变**——这是本周最重要的认知收获。线性回归本质上就是一个加权求和公式，隐含了"每个特征独立且线性地影响目标"的强假设。MLP 通过 ReLU 激活函数和层级结构，能够学习到特征之间的交互关系（比如"戴口罩比例高**且**确诊数在下降趋势中"→ 确诊数偏低）。

### 2.6 过拟合与正则化

| 技术 | 原理 | 在本项目中的效果 |
|---|---|---|
| **Dropout(0.2)** | 每次训练随机丢弃 20% 神经元，迫使各个神经元独立有用 | 原始 MLP 在 200~500 轮验证 RMSE 保持稳定波动，未出现严重过拟合 |
| **BatchNorm 的隐含正则化** | mini-batch 统计量的噪声相当于随机扰动 | BN MLP 的验证 RMSE 波动比原始 MLP 更小 |

## 三、遇到的主要问题与解决

### 问题 1：验证集划分方式导致 RMSE 差异 18.8%

**现象**：同一线性模型，"随机划分"验证 RMSE 为 7.29，"i%10 等间隔划分"验证 RMSE 为 5.92。同样的模型、同样的数据，仅因划分方式不同差异就接近 19%。

**分析**：
- 数据按时间排序，等间隔采样的验证集样本在时间上与训练集交织在一起，预测难度更低
- 等间隔划分给了模型更多训练数据（2430 vs 2160）

**解决**：理解到验证 RMSE 不能作为绝对标准，划分方式直接影响评估结果。后续实验统一使用随机 80%/20% 划分。相关分析记录在 `test_report.md`。

### 问题 2：线性回归预测出现负值

**现象**：线性回归预测值最小为 -8.17（合理范围 ≥0）。

**分析**：`y = w₁x₁ + ... + b` 对所有输入不加约束，特定特征组合可能导致负输出。线性回归只有数学计算没有物理常识。

**解决**：MLP 虽然也不能严格保证正值，但多层非线性结构使输出范围更合理（3.35 ~ 41.45）。

### 问题 3：BatchNorm 效果提升不明显（提升仅 1%）

**现象**：BN MLP 比原始 MLP 验证 RMSE 仅提升 0.0109（0.9965 vs 1.0074）。

**分析**：
- 输入已做 Z-score 归一化，缓解了输入层级的尺度问题
- 网络只有 2 个隐藏层，内部协变量偏移不严重
- BN 的优势在更深的网络中才会充分体现

**收获**：不是所有技巧在所有场景都有效——重要的是理解"为什么效果不显著"，这个分析过程本身就是重要的学习。

### 维度 5：关键发现总结

1. **验证集划分方式直接决定评估可信度**：同一模型、同一数据，仅划分方式不同 → RMSE 差 18.8%。**验证 RMSE 不是绝对标准**，后续实验必须固定划分策略。

2. **线性回归到 MLP 是质变，不是量变**：RMSE 从 7.29 降至 0.98（↓86.5%），参数量从 94 增至 20,353（×216）。但 MLP 仅用 50 轮就超越了线性回归 300 轮的效果——这背后是"表达能力"而非"参数量"的差距。

3. **BatchNorm 让 MLP 更稳定，而非更精确**：BN 的真实价值在"收敛速度 +30 轮、验证波动 −38%、泛化更好"，最终 RMSE 的提升（↓1.08%）反而是次要的。

4. **Dropout 控制过拟合有效**：MLP 在 200~500 轮中训练 RMSE 持续下降但验证 RMSE 未恶化，验证了 `Dropout(0.2)` 的正则化效果。

5. **浅层网络中，输入归一化是最关键的一步**：输入 Z-score 归一化已帮模型解决了输入尺度的最大问题，BN 的增量收益在小网络上有限。这个"提升不明显的分析"本身也是研究成果。

### 维度 6：各模型对任务的适配性分析

| 模型 | 输出是否存在负值 | 预测值分布合理性 | 任务适配性 |
|---|---|---|---|
| 线性回归 | **-8.17**（存在，不合理）| 较差（31.53 上限偏低）| ❌ 不适配 |
| MLP（原始） | 3.35（合理，≥0）| 41.73 上限更接近真实分布 | ✓ 适配 |
| MLP（BatchNorm） | 3.45（合理，≥0）| 42.13 上限接近 MLP | ✓ 适配 |

线性回归的负值预测（-8.17）进一步证实了非线性模型在此任务上的必要性。
