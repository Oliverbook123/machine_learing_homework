# 测试研究报告：线性回归 vs MLP 对比研究

## 1. 实验目的

在新冠每日确诊病例预测任务中，对比线性回归与 MLP（多层感知机神经网络）两种模型的效果差异，探究以下问题：

1. 引入非线性能力（MLP）对回归任务是否有提升，提升有多大？
2. 线性模型和神经网络在预测行为上有哪些不同（如预测值分布、过拟合程度等）？
3. 两者为什么会有差异？

## 2. 实验设置

| 项目 | 线性回归 | MLP |
|---|---|---|
| 模型结构 | `Linear(93 → 1)` | `Linear(93→128) → ReLU → Dropout(0.2) → Linear(128→64) → ReLU → Dropout(0.2) → Linear(64→1)` |
| 参数量 | 94 | 20,353 |
| 模型本质 | `y = w₁x₁ + w₂x₂ + ... + w₉₃x₉₃ + b` | 多层线性变换 + 非线性激活 |
| 非线性能力 | 无 | 有（ReLU 激活函数） |
| 防止过拟合 | 不涉及（模型太简单） | Dropout(0.2) |
| Batch size | 64 | 64 |
| 学习率 | 1e-3 | 1e-3 |
| 优化器 | Adam | Adam |
| 训练轮数 | 300 | 500 |
| 训练/验证集 | 2160 / 540（随机 80%/20%） | 2160 / 540（随机 80%/20%） |
| 损失函数 | MSELoss | MSELoss |
| 评估指标 | RMSE | RMSE |

## 4. 结果分析

### 4.1 学习效率的巨大差距

线性回归在第 50 轮时 Train RMSE 仍高达 14.80；MLP 在第 50 轮时验证 RMSE 已经降到 1.13。**MLP 仅用 50 轮就远远超越了线性回归 300 轮的效果。**

这说明 MLP 能够更快地从数据中发现有效模式。线性回归受限于 "特征之间只能线性组合" 的约束，需要更多轮次来缓慢逼近——但即使逼近得再好，上限也很有限。

### 4.2 模型容量 vs 拟合能力

| 模型 | 参数量 | 表达能力 | 现象 |
|---|---|---|---|
| 线性回归 | 94 | 只能学习线性关系 | 300 轮仍未收敛，提升空间有限 |
| MLP | 20,353 | 可学习复杂非线性关系 | 200 轮后性能趋于平稳，Dropout 有效控制了过拟合 |

线性回归本质上就是一个大型的 **加权求和公式**：

```
确诊 = w₁×戴口罩 + w₂×出门 + w₃×焦虑 + ... + w₉₃×特征₉₃ + b
```

这隐含了一个强假设：每个特征对确诊数的影响是 **独立且线性的**。但现实中很可能不是这样——比如"戴口罩比例从 50% 提高到 60%"和"从 90% 提高到 95%"，对确诊数的影响可能截然不同（边际效应递减）。

MLP 通过多层结构和 ReLU 激活函数，能够捕捉到这类复杂关系。

### 4.3 过拟合与 Dropout 的作用

MLP 在 200~500 轮之间的验证 RMSE 表现如下：

```
轮次    训练 RMSE      验证 RMSE
200      1.74           1.03  ← 最佳验证点
250      1.71           1.10
300      1.61           1.05
350      1.63           1.12
400      1.56           1.02
450      1.59           1.04
500      1.49           1.18
```

从数据中可以看出：

1. **过拟合并不严重**。训练 RMSE 从 1.74 缓慢下降到 1.49，降幅只有约 14%。验证 RMSE 在 0.98 ~ 1.18 之间波动，没有出现"训练集越来越好、验证集越来越差"的经典过拟合趋势。

2. **Dropout(0.2) 起到了关键作用**。Dropout 在每次训练迭代中随机丢弃 20% 的神经元，迫使每个神经元都学到"独立有用"的特征，而不是依赖某些特定神经元。这种**正则化**策略有效地抑制了过拟合。

3. **波动而非恶化**。200 轮之后验证 RMSE 的波动（1.02 ← 1.10 → 1.05 → 1.12 → 1.02）更多是**随机波动**，不是趋势性退化。这说明模型已经趋于收敛，在最优解附近振荡。

4. **仍然需要 Early Stopping**。虽然过拟合不明显，但 200~500 轮的训练并没有带来实质提升。加入 Early Stopping 可以在验证 RMSE 到达平台期后自动停止训练，节省计算资源。

这个现象告诉我们：**Dropout 是防止过拟合的有效手段**。但即使过拟合不严重，无节制地增加训练轮数也不会持续提升效果——模型最终会到达一个平台期。

### 4.4 预测合理性对比

线性回归预测出 -8.17 的确诊人数，这是**物理上不可能的值**（确诊人数不可能为负）。

原因：线性回归的公式 `y = w₁x₁ + w₂x₂ + ... + b` 对所有输入都是无限制的。当输入特征取某些值组合时，预测值自然可能落入负数范围。线性回归**没有"常识"**，它只是在做数学运算。

MLP 虽然也不能保证预测一定为正，但多层非线性结构使得它的输出范围更加自然，本次实验中预测值全部为正数（3.35 ~ 41.45）。

### 4.5 参数量与计算成本

| 模型 | 参数量 | 训练时间 | 是否划算 |
|---|---|---|---|
| 线性回归 | 94 | 快 | 简单任务够用 |
| MLP | 20,353（×216 倍） | 较慢 | 复杂任务值得 |

MLP 参数量是线性回归的 **216 倍**，训练速度更慢。但考虑到 RMSE 从 7.29 降到 0.98，性能提升 86.5%，这个代价是完全值得的。

## 5. 为什么 MLP 效果更好？— 对数据的理解

### 5.1 数据结构的线索

数据集包含三个连续时间窗口的相同特征，比如：

- 第 1 天：戴口罩比例 84%，确诊 0.84
- 第 2 天：戴口罩比例 83%，确诊 0.90
- **第 3 天：戴口罩比例 84%，确诊 ？（要预测）**

线性回归的做法：`确诊₃ = w₁×口罩₁ + w₂×口罩₂ + w₃×口罩₃ + ...`，每个特征独立加权。

但实际上，**"第 1 天确诊 0.84 → 第 2 天确诊 0.90"** 这个变化趋势本身可能就是一个重要信号——确诊数在上升，说明疫情在恶化。线性回归不擅长捕捉这种"变化模式"，而 MLP 的多层结构可以。

### 5.2 特征交互

现实中可能存在这样的关系：
- "戴口罩比例较高 **且** 确诊数下降" → 第 3 天确诊数偏低
- "焦虑比例上升 **且** 确诊数上升" → 第 3 天确诊数偏高

这需要模型能学习到 **特征之间的交互（Interaction）**。线性回归无法做到（因为各个特征独立加权），MLP 则可以。

## 6. 结论与建议

### 6.1 核心结论

1. **MLP 在本任务中全面碾压线性回归**，验证 RMSE 从 7.29 降到 0.98，降幅达 86.5%
2. **线性回归的预测不合理**，出现了负的确诊人数
3. **MLP 未出现明显的过拟合**，Dropout(0.2) 有效地抑制了过拟合。验证 RMSE 在 200~500 轮之间保持平稳波动，但无节制地增加训练轮数并不会持续提升效果
4. **数据中存在显著的非线性关系**，简单的线性加权无法充分拟合

### 6.2 改进建议

| 改进点 | 当前状态 | 建议 |
|---|---|---|
| Early Stopping | 无 | 验证损失连续 N 轮不下降就停训 |
| L2 正则化 | 无 | Adam 加 `weight_decay=1e-4`，进一步防止过拟合 |
| 学习率调度 | 固定 1e-3 | 用学习率衰减，后期用小学习率精细调参 |
| 特征选择 | 全用 93 列 | 参考大佬只取州编码 + 前两天的 tested_positive，看 RMSE 能否继续降 |
| 验证集划分 | 随机 80%/20% | 改为 i%10 确定性划分，保证结果可复现 |

### 6.3 一句话总结

> **从线性回归到 MLP 不是量变，是质变。** 非线性能力让模型从 94 个参数变成 20,353 个参数，也让 RMSE 从 7.29 降到了 0.98。代价是更容易过拟合——所以 MLP 需要更多的"规矩"（Dropout、Early Stopping、正则化）来管住它。
