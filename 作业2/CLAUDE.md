## 2-1 音素分类 (Phoneme Classification)

### 任务介绍
- **任务**：多分类问题 (Multiclass Classification)
- 从语音中进行逐帧的音素预测。
- **什么是音素？**
  语言中语音声音的基本单位，可以用来区分不同的单词。
  - 例如：bat / pat, bad / bed
  - 机器学习 (Machine Learning) → 发音为：M AH SH IH N L ER N IH NG

### 数据预处理与声学特征
- **特征提取**：39维 MFCC（梅尔频率倒谱系数）或 80维滤波器组等。每一帧（frame）会被转换为一个向量。
- **声学特征 - MFCC**：更多详情请参考林善山教授的《数字语音处理导论》第7章。（图片来源：李宏毅教授 2020春季 DLHLP 语音识别课程）

### 关于数据的更多信息
由于每帧仅包含 25 毫秒的语音，单帧不太可能代表一个完整的音素：
- 通常，一个音素会跨越多个帧。（提示：后处理可能会有所帮助）
- **拼接相邻帧进行训练**：
  - 在本次作业中，我们将过去和未来的各 5 帧拼接起来进行训练（总共 11 帧）。
  - 你可以将输入形状 `(1, 429)` 重塑 (reshape) 为 `(11, 39)`，以分离出 11 帧。
  - 请记住，**标签对应的是中心帧**。
- **严禁**寻找测试集标签或进行人工标注！

### 数据集与数据格式
- **数据集**：TIMIT 声学-语音连续语料库（针对英语进行了语音平衡）。
- **数据格式**（助教已预处理完毕）：
  - `timit_11/`
    - `train_11.npy` → 训练数据 (训练帧数, 11 x 特征维度)
    - `train_label_11.npy` → 逐帧音素标签 (0-38)
    - `test_11.npy` → 测试数据 (测试帧数, 11 x 特征维度)
- **声学特征** (39维 MFCC)：拼接前后各5帧（特征维度 = 11 x 39 = 429），每个输入的音素标签对应中心帧。
- **禁止使用额外数据**，否则你的最终成绩将乘以 0.9！
- **类别映射**：共 39 个类别（0-38），包括各种元音、辅音及静音/闭合音（如：0=iy, 13=l, 26=dx, 38=sil 等）。

## 2-2 黑塞矩阵 (Hessian Matrix)

### 任务介绍
- **任务**：黑塞矩阵 (Hessian Matrix)
- 假设我们正在训练一个神经网络，并试图找出模型是否达到了**局部极小值类点 (local minima-like point)**、**鞍点 (saddle point)** 或**以上皆非**。我们可以通过计算黑塞矩阵来做出判断。
- **什么是黑塞矩阵？** 它是模型的二阶偏导数。强烈建议在开始此部分前观看课程视频。
- 本任务的目标函数是一个单变量 sinc 函数。你将获得：
  - 助教训练好的模型权重 (checkpoint)
  - 一批训练数据
  - 一个损失函数
- 你需要计算黑塞矩阵并据此做出判断。

### 梯度范数 / 极小值比例
1. **梯度范数 (Gradient Norm)**：在正常的训练过程中，梯度很少恰好等于零。在本次作业中，我们将**梯度范数小于 `1e-3`** 视为零。
2. **极小值比例 (Minimum Ratio)**：对于理想的局部极小值，黑塞矩阵的所有特征值都大于零。我们将正特征值的比例定义为“极小值比例”。
   - 在本次作业中，如果**极小值比例 > 0.5 且 梯度范数 < 1e-3**，我们假设模型处于“局部极小值类点”。

**判断规则：**
- 梯度范数 < 1e-3 且 极小值比例 > 0.5 => **局部极小值类点 (local minima like)**
- 梯度范数 < 1e-3 且 极小值比例 <= 0.5 => **鞍点 (saddle point)**
- 梯度范数 >= 1e-3 => **以上皆非 (none of the above)**
