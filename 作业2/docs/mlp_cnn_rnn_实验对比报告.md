# MLP / CNN / RNN 音素分类实验对比报告

## 一、实验概述

### 1.1 任务说明

音素分类：输入 429 维 MFCC 特征（11 帧 × 39 维），预测每帧对应的音素类别（0-38，共 39 类）。

### 1.2 数据集

| 数据集 | 样本数 | 特征维度 | 标签范围 |
|--------|--------|---------|---------|
| 训练集 | 1,106,938（90%） | 429 | 0-38 |
| 验证集 | 122,994（10%） | 429 | 0-38 |
| 测试集 | 451,552 | 429 | 无标签 |

### 1.3 统一配置（三个模型共享）

| 超参数 | 数值 |
|--------|------|
| Batch size | 256 |
| 验证集比例 | 10% |
| 优化器 | Adam |
| 初始学习率 | 0.001 |
| 学习率调度 | CosineAnnealingLR |
| Weight decay | 1e-4 |
| Dropout | 0.3 |
| Epochs | MLP=12, CNN=20, RNN=20 |

## 二、模型架构对比

### 2.1 MLP（多层感知机）

**输入处理方式：** 直接将 429 维向量送入全连接层，**逐帧独立预测**。

**网络结构：**
```
输入 (batch, 429)
  → Linear(429→512) → BN → ReLU → Dropout(0.3)
  → Linear(512→256) → BN → ReLU → Dropout(0.3)
  → Linear(256→128) → BN → ReLU → Dropout(0.3)
  → Linear(128→39)
```

**参数量：** ~389,415

**核心特点：** 不利用帧间时序关系，每帧独立做决策。

### 2.2 CNN（卷积神经网络）

**输入处理方式：** 429 维 → reshape 为 (39, 11)，39 种 MFCC 系数作为通道，11 帧作为时间轴。用 Conv1d 沿时间维度提取局部帧间模式。

**网络结构：**
```
输入 (batch, 429) → reshape → (batch, 39, 11)
  → Conv1d(39→128, k=3, p=1) → BN → ReLU
  → Conv1d(128→256, k=3, p=1) → BN → ReLU
  → AdaptiveAvgPool1d(1) → flatten → (256)
  → Linear(256→128) → BN → ReLU → Dropout(0.3)
  → Linear(128→39)
```

**参数量：** ~281,191

**核心特点：** 用卷积核看相邻 3 帧的局部模式，但感受野有限。

### 2.3 RNN（LSTM——长短时记忆网络）

**输入处理方式：** 429 维 → reshape 为 (11, 39)，视为 11 个时间步的序列，每步输入一帧的 39 维 MFCC。LSTM 依次处理各帧，用隐藏状态传递帧间信息。

**网络结构：**
```
输入 (batch, 429) → reshape → (batch, 11, 39)
  → LSTM(39→128, 2层, 双向) → (batch, 11, 256)
  → 对 11 帧输出取均值池化 → (256)
  → Linear(256→128) → BN → ReLU → Dropout(0.3)
  → Linear(128→39)
```

**参数量：** ~321,319

**核心特点：** 双向 LSTM 建模整个 11 帧序列的正反向依赖，每个帧的表示都包含前后文信息。

## 四、结果分析

### 4.1 RNN（LSTM）表现最佳的原因

| 维度 | RNN | 分析 |
|------|-----|------|
| 最佳 Val Acc | **0.7411** | 三者最高，接近 reference MLP(0.7508) |
| 收敛速度 | epoch 1 就达 0.7148 | 远超 MLP(0.6693) 和 CNN(0.6510) |
| 参数量 | 321K（中等） | 用最少的参数实现了最高的 acc/param 效率 |

**核心原因：** 音素本身具有时序连续性（一个音素通常跨越多个帧）。双向 LSTM 能捕捉**整个 11 帧的上下文依赖**，这是 MLP（逐帧独立）和 CNN（只看局部 k=3）都做不到的。

**过拟合问题：** RNN 的 Val Loss 从 epoch 4 的 0.8147 一路攀升到 epoch 20 的 0.9706，而 Train Loss 却持续下降到 0.3974——典型的过拟合迹象。说明 RNN 的记忆能力太强，开始记住训练数据的具体模式而非泛化特征。如果增加正则化或早停，RNN 的 Val Acc 有望更接近 reference MLP 的 0.7508。

### 4.2 CNN 表现最差的可能原因

| 维度 | CNN | 分析 |
|------|-----|------|
| 最佳 Val Acc | **0.7024** | 三者最低 |
| 收敛速度 | 慢，epoch 20 仍在缓慢上升 | 20 epoch 不够充分收敛 |

**可能原因：**

1. **时间维度太短（11 帧）：** Conv1d 在长度=11 的序列上做 k=3 的卷积，每层卷积只能看到相邻 3 帧。经过两层堆叠后感受野也只有 5 帧，无法像双向 LSTM 那样看到全部 11 帧的上下文。

2. **池化损失信息：** AdaptiveAvgPool1d(1) 将 11 帧的信息暴力压缩为 1 个值，可能丢失了关键帧（如中心帧）的细节。

3. **通道设计不自然：** 将 39 种 MFCC 系数作为通道，假设了不同 MFCC 系数之间是独立的——但 MFCC 系数本身在频率轴上是连续的，相邻 MFCC 有相关性。更自然的做法是 Conv2d 同时卷积时间和频率轴。

### 4.3 MLP 表现居中

MLP 的 0.7269 介于 RNN 和 CNN 之间。它是三者中参数量最大的（389K），但由于逐帧独立处理、完全不利用时序信息，上限受到限制。

但与 RNN 相比，MLP 的 Val Loss 一直在改善（0.8366 在 epoch 12），而 RNN 早已开始过拟合。这说明 MLP 虽然上限低，但更稳定。

## 六、结论与建议

### 6.1 主要结论

1. **RNN（LSTM）最适合此任务**，因为音素分类本质上是时序依赖问题，LSTM 的双向建模能力天然匹配。

2. **CNN 在此任务上表现不佳**，主要是因为 11 帧的时域太短，卷积和池化操作损失了关键信息。

3. **MLP 稳健但上限有限**，逐帧独立预测的假设在音素分类这个时序任务上不成立。

4. **RNN 的过拟合问题突出**，需要更强的正则化（更大的 weight_decay、更小的 Dropout、早停等）。

### 6.2 后续优化方向

| 方向 | 建议 | 预期效果 |
|------|------|---------|
| RNN 正则化 | 增加 Dropout 到 0.5，增大 weight_decay | 缓解过拟合，可能提升 0.5-1% |
| CNN 改进 | 改用 Conv2d（同时卷时间和频率），或增大核尺寸 | 可能显著提升 |
| 后处理 | 三个模型都加滑动窗口平滑（如 mlp_phoneme 所做） | 测试集可能提升 |
| 模型集成 | 平均 MLP + RNN 的预测结果 | 利用两者不同的偏差 |
